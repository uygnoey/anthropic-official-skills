## 이 글이 뭔가요
이 글은 Claude Developer Platform에서 장시간 실행되는 에이전트를 위해 제공되는 두 가지 기능을 소개합니다. (1) 컨텍스트 편집: 토큰 한도에 가까워지면 오래된 도구 호출/결과를 자동으로 정리하고, (2) 메모리 도구: 컨텍스트 창 밖에 파일 기반 저장소로 정보를 보존(개발자가 저장 위치를 제어)합니다.

## 언제 유용한가요
여러 단계로 오래 실행되거나 세션을 넘나드는 에이전트를 만들 때, 컨텍스트 소진을 막으면서도 중요한 결정/발견/상태를 턴 간에 유지하고 싶을 때 유용합니다.

## 핵심 포인트
- 컨텍스트 편집은 토큰 한도 근처에서 오래된 도구 호출/결과를 제거하면서 대화 흐름을 유지합니다.
- 메모리 도구는 개발자가 관리하는 전용 디렉터리에 중요한 정보를 저장해 다음 세션에서도 참조할 수 있게 합니다(도구 호출을 통한 클라이언트 측 동작).
- 글에서는 Claude Sonnet 4.5가 이러한 컨텍스트 인지 워크플로를 개선한다고 설명합니다.

## 번들 리소스
- Skill: 컨텍스트에 남겨둘 것과 영속 메모리에 기록할 것을 구분하는 실전 플레이북, 그리고 “보관 vs 유지” 판단을 돕는 체크리스트.
