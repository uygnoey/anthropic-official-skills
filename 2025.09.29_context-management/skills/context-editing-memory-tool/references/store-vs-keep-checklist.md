# Store vs keep checklist

Use this checklist when deciding whether information should stay in the context window or be written to persistent memory.

## Write to memory (store)
- Decisions that affect future work (architecture, chosen approach, trade-offs)
- Constraints and “do not do” rules
- Definitions and naming conventions
- Confirmed findings and their sources
- Partial progress that must survive interruptions (what was tried, what failed, what to try next)

## Keep in context (keep)
- The active task goal and acceptance criteria
- The current plan for the next few steps
- The minimal set of tool outputs needed to act next
- Temporary data that can be regenerated cheaply
