# Skillsを活用してフロントエンドデザインを改善する

## この記事について
この記事では、Claudeの**Skills**を再利用可能なオンデマンドのデザインガイダンスとして活用し、フロントエンド/UI生成の結果が「AIのデフォルト」（よくあるフォントや予測しやすいグラデーションなど）に収束してしまう問題を解消する方法を説明します。

## どんなときに役立つか
- ランディングページ、ダッシュボード、アプリなど顧客向けの体験において、ブランドらしいUIを生成したいとき（汎用レイアウトは不要）。
- デザインルールを再利用可能な形で保持しつつ、ベースプロンプトを肥大化させたくないとき。
- 単一ファイルの簡易サンプルではなく、React + Tailwind + コンポーネントライブラリといったモダンなスタックでWebアーティファクトを生成したいとき。

## 主なポイント
- Skillsはファイルとして保存しておき、必要なときだけClaudeに読み込ませることができるため、専門的なデザイン制約を通常のプロンプトから切り離して管理できる。
- 指示は「適切な粒度」で行う。抽象的すぎる指示も、ピクセルやhex値レベルの細かすぎる指定も、どちらも問題がある。
- タイポグラフィ、テーマ、モーション、背景の選択において、よくあるデフォルトを避けるために具体的な代替案を示すと、出力品質が向上する。
- 記事にはタイポグラフィ選択のプロンプトブロック、RPGテーマ、そしてより包括的な`frontend_aesthetics`ブロックの例が含まれている。
