## ¿De qué trata este post?
Este post cuenta cómo tres startups de Y Combinator usan Claude Code para enviar producto más rápido, gestionar el contexto de forma deliberada y escalar flujos de trabajo de desarrollo agente.

## ¿Cuándo es útil?
Es útil cuando necesitas un método repetible para separar trabajo en sesiones de investigación/planificación/implementación, o cuando estás evaluando flujos con subagentes para investigar en paralelo.

## Puntos clave
- Las startups describen un flujo en tres fases: investigación, planificación e implementación (a menudo en sesiones separadas).
- La gestión del contexto es clave: inicia una nueva sesión cuando el contexto se vuelve contradictorio o ruidoso.
- Los subagentes pueden paralelizar la investigación del codebase y trabajar por tipo de datos.
