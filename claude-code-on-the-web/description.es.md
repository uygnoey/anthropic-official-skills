# ¿De qué trata esta habilidad?
Esta habilidad te ayuda a delegar tareas de programación desde un entorno de trabajo basado en navegador conectado a GitHub, donde cada tarea se ejecuta en un entorno cloud aislado y puede generar una pull request.

# ¿Cuándo es útil?
- Cuando tienes correcciones de errores bien definidas o tareas de mantenimiento rutinario del código.
- Cuando quieres ejecutar múltiples tareas de programación en paralelo en distintos repositorios.
- Cuando prefieres no usar una terminal local y quieres hacer seguimiento del progreso desde el navegador.

# Principios fundamentales
- Mantén las tareas bien acotadas y verificables (criterios de aceptación claros).
- Valida los cambios (tests, linting) antes de abrir una PR.
- Usa el "steering": ajusta los requisitos mientras la tarea está en curso.

# Ejemplos
- Corregir una suite de pruebas que falla y abrir una PR.
- Resolver un backlog de issues pequeños como PRs separadas.

# Lista de verificación / consejos
- Proporciona el repositorio, la rama y los criterios de éxito.
- Prioriza las correcciones rutinarias frente al trabajo nuevo y ambiguo.
