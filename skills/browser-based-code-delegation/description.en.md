# What is this skill?
This skill helps you delegate coding tasks from a browser-based workspace connected to GitHub, where each task runs in an isolated cloud environment and can produce a pull request.

# When should I use it?
- When you have well-defined bugfixes or routine code maintenance.
- When you want to run multiple coding tasks in parallel across repositories.
- When you prefer not to use a local terminal and want progress tracking in the browser.

# Core principles
- Keep tasks scoped and testable (clear acceptance criteria).
- Validate changes (tests, linting) before opening a PR.
- Use steering: adjust requirements while the task is in progress.

# Examples
- Fix a failing test suite and open a PR.
- Tackle a backlog of small issues as separate PRs.

# Checklist / tips
- Provide the repo + branch + success criteria.
- Prefer routine fixes over ambiguous greenfield work.
