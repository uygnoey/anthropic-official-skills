## 이 글이 뭔가요
이 글은 Y Combinator 스타트업 3곳이 Claude Code를 활용해 개발 속도를 높이고, 컨텍스트를 의도적으로 관리하며, 에이전틱 개발 워크플로를 확장하는 방법을 소개합니다.

## 언제 유용한가요
리서치/계획/구현을 세션 단위로 분리하는 반복 가능한 방식이 필요하거나, 병렬 조사를 위한 서브에이전트 워크플로를 고려할 때 유용합니다.

## 핵심 포인트
- 3단계 워크플로(리서치 → 계획 → 구현)를 강조하며, 종종 각 단계를 별도 세션으로 운영합니다.
- 컨텍스트 관리가 중요하며, 모순되거나 노이즈가 쌓이면 새 세션을 시작합니다.
- 서브에이전트 워크플로로 코드베이스 조사를 병렬화하고 데이터 타입별 작업을 분리할 수 있습니다.
