# 가이드: Claude Code에서 Opus 4.7 활용 모범 사례

이 가이드는 Claude Code에서 프롬프트와 세션을 구성할 때 적용할 수 있는 체크리스트 형태로 포스트의 권장 사항을 정리한 것입니다.

## 대화형 세션 구조화
- Claude를 줄 단위로 지시하는 페어 프로그래머보다는, 업무를 위임할 수 있는 유능한 엔지니어처럼 대하세요.
- 첫 번째 턴에서 의도, 제약 조건, 완료 기준, 관련 파일 위치 등 작업 내용을 미리 명확하게 지정하세요.
- 질문을 묶어서 전달하고 사용자 상호작용 횟수를 줄여 불필요한 추론 오버헤드를 방지하세요.
- 모델이 적은 체크인으로도 안전하게 실행할 수 있다고 신뢰할 수 있는 장시간 작업에는 오토 모드를 적절히 활용하세요.

## 노력 수준 가이드
- `xhigh`는 Claude Code에서 Opus 4.7의 기본 노력 수준으로, `high`와 `max` 사이에 해당합니다.
- 비용을 줄이거나 동시 세션을 실행할 때는 `high`를 사용하세요.
- `max`는 진정으로 어려운 문제에 한해 신중하게 사용하세요. 수익 체감과 과잉 사고 가능성이 높아집니다.

## 적응형 사고 프롬프트
- Opus 4.7은 고정된 사고 예산이 있는 Extended Thinking 대신 적응형 사고를 사용합니다.
- 더 많은 사고를 유도하려면, 응답 전에 신중하고 단계적인 추론을 요청하세요.
- 사고를 줄이려면, 모델에게 확신이 없을 때 빠르고 직접적으로 응답하도록 요청하세요.

## 고려해야 할 동작 변화
- 응답 길이는 작업의 복잡도에 맞게 조정됩니다. 특정 형식이나 길이가 필요하다면 명시적으로 지정하세요.
- 모델은 도구 호출 빈도를 줄이고 추론을 더 많이 합니다. 도구 사용을 원할 경우 언제, 왜 사용해야 하는지 명시적으로 설명하세요.
- 기본적으로 서브에이전트 생성 수가 줄어듭니다. 파일이나 독립적인 항목에 대해 병렬 처리가 필요할 때는 명시적으로 병렬 서브에이전트를 요청하세요.
