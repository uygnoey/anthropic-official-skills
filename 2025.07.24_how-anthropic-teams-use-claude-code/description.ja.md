# AnthropicのチームがClaude Codeを使う方法

## この記事について
本記事は、Anthropicの複数チームがClaude Codeをコードベースの把握、テスト、デバッグ、プロトタイピング、ドキュメント作成、ワークフロー自動化に活用している具体例を紹介します。

## どんなときに役立つか
- 未知のコードベースにオンボーディングし、依存関係の理解を素早く進めたいとき。
- デバッグ、インシデント対応、テスト作成、迅速なプロトタイプ開発を加速したいとき。
- 大規模CSVの分析や広告文の大量生成など、反復的な作業を自動化したい／専門化したサブエージェントが有効なとき。

## 主なポイント
- リポジトリ（CLAUDE.mdを含む）を読み、関連ファイルを特定し、依存関係やデータパイプラインを説明する用途。
- 包括的なテストスイートの生成・リファクタリングに使われ、場合によってはPRの自動フィードバックにも統合。
- インシデント時はスタックトレースとドキュメントを与え、制御フロー追跡と対策提案に利用。
- 機能開発では（コード作成→テスト実行→反復）の自律ループで開発し、エッジケースを探索。
- 複数の情報源を統合し、Markdownのランブックを作成。
- 制約付きの大量生成（例：広告バリエーション）には、複数の専門サブエージェントを使うケースがある。
- デザインのフレームから広告バリエーションを生成するFigmaプラグイン事例も言及されます。
