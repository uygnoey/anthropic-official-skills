# Anthropic 팀들이 Claude Code를 사용하는 방법

## 이 글이 뭔가요
이 글은 Anthropic 내부의 여러 팀이 Claude Code를 코드베이스 탐색, 테스트, 디버깅, 프로토타이핑, 문서화, 자동화에 활용하는 구체적 사례를 소개합니다.

## 언제 유용한가요
- 익숙하지 않은 코드베이스에 온보딩하며 의존성/구조를 빠르게 파악해야 할 때.
- 디버깅, 인시던트 대응, 테스트 작성, 빠른 프로토타이핑 속도를 높이고 싶을 때.
- 대규모 CSV 분석이나 광고 문구 대량 생성처럼 반복 작업이 많고, 자동화 또는 전문화된 서브에이전트가 도움이 될 때.

## 핵심 포인트
- 팀들은 Claude Code가 리포지토리(및 CLAUDE.md)를 읽고 관련 파일을 찾아 의존성과 데이터 파이프라인을 설명하도록 사용합니다.
- 포괄적인 테스트 스위트 생성/리팩터링에 활용하며, 경우에 따라 PR 자동 피드백 루프에도 통합합니다.
- 인시던트 상황에서는 스택 트레이스와 문서를 제공해 제어 흐름을 추적하고 완화책을 제안받습니다.
- 기능 개발에서는 (코드 작성 → 테스트 실행 → 반복) 형태의 자율 루프를 돌려 엣지 케이스를 탐색합니다.
- 여러 문서 소스를 종합해 마크다운 런북을 만드는 데도 사용합니다.
- 일부 워크플로는 여러 전문 서브에이전트를 통해 제약 조건을 만족하는 대량 결과물(예: 광고 변형)을 빠르게 생성합니다.
- 글에는 디자인 프레임에서 광고 변형을 생성하는 Figma 플러그인 사례도 언급됩니다.
