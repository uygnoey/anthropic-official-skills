# Extending Claude's capabilities with skills and MCP servers

## ¿De qué trata este post?
Este post explica cómo las skills y los servidores de Model Context Protocol (MCP) se complementan al construir agentes que siguen los flujos de trabajo de tu equipo.

## ¿Cuándo es útil?
Es útil cuando decides qué capturar como conocimiento procedimental reutilizable (skills) versus qué conectar como acceso en vivo a herramientas (MCP), especialmente en flujos de trabajo de varios pasos.

## Puntos clave
- MCP aporta conectividad (acceso estandarizado y seguro a sistemas externos) y las skills aportan experiencia (lógica del flujo y estándares).
- Las skills sirven para flujos de trabajo de varios pasos, consistencia, experiencia de dominio y conocimiento institucional.
- MCP sirve para acceso a datos en tiempo real, acciones en sistemas externos, operaciones de archivos e integraciones de API.
- Al combinarlos, evita instrucciones en conflicto (p. ej., MCP pide JSON y la skill pide tablas en Markdown).
