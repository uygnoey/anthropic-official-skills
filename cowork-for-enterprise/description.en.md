## What is this post?
This post announces that Claude Cowork is generally available on all paid plans and describes enterprise-ready admin controls, observability, and connectors.

## When is it useful?
Use it if you administer Claude Cowork in a company and need governance features like role-based access, spend controls, usage analytics, and observability for tool/connector activity.

## Key points
- Role-based access controls for Enterprise: organize users into groups (including via SCIM) and assign custom roles that define capabilities.
- Group spend limits: set per-team budgets.
- Usage analytics: admin dashboard metrics plus an Analytics API (including per-user activity and skill/connector invocations).
- Expanded OpenTelemetry events for tool/connector calls, files read/modified, skills used, and action approval signals.
- Zoom MCP connector for bringing meeting summaries/transcripts into Cowork workflows.
- Per-connector action controls: allow read access but disable write (or similar) org-wide.
