## この記事について
この記事は、Claude Code チームがモデルの視点から考える（「エージェントとして見る」）ことで、エージェント用ツールを設計・検証・改善していく方法を解説します。

## どんなときに役立つか
エージェント向けツール（またはエージェント基盤）を作る際に、新しいツールを追加すべきか、インターフェースをどう設計すべきか、選択肢を増やしすぎてモデルを混乱させないためにどうするかを検討するときに役立ちます。

## 主なポイント
- ツールのインターフェースはモデルの現在の能力に合わせ、能力の変化に応じて前提を見直します。
- ツール追加にはコストがあり、選択肢が増えるほど意思決定負荷が上がります。
- 目的特化ツールを増やすより、progressive disclosure（段階的な公開）や再利用可能なスキル文書を優先します。
- ツールがボトルネックになったら、より表現力の高いプリミティブに置き換えます（例: todo から task 抽象へ）。
- ユーザーへの確認は、別ツールを無理に拡張するより、専用の質問ツールのほうが安定する場合があります。
