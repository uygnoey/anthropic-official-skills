# Notes (from the source post)

## Triggering behavior

- The **name** and **description** are what influence triggering.
- Triggering is semantic, not simple keyword matching.
- Multiple skills can activate simultaneously for complex requests.

## Context size guidance

- Avoid bloating the context window.
- Use companion files as a “menu” so only necessary details are loaded.

## Testing and iteration

- Test with normal, edge-case, and out-of-scope requests.
- Iterate based on real-world usage.
