# Build and share AI-powered apps with Claude

## ¿De qué trata este post?
Este post presenta los ‘Claude-powered artifacts’: apps interactivas dentro de Claude que pueden llamar a Claude mediante una API y compartirse de modo que cada usuario se autentica con su propia cuenta de Claude y el uso se descuenta de su suscripción (sin gestionar claves API).

## ¿Cuándo es útil?
Útil cuando quieres prototipar y compartir rápidamente una experiencia interactiva con IA dentro de Claude sin infraestructura, o cuando necesitas explicar a tu equipo cómo funcionan el uso compartido y la facturación.

## Puntos clave
- Los artifacts pueden convertirse en apps interactivas que llaman a Claude mediante una API desde Claude.
- Al compartir, el visor usa su propia cuenta de Claude: su consumo cuenta contra su suscripción; el creador no paga por su uso; no hacen falta claves API.
- Claude genera código real, ayuda a depurarlo y cualquier persona puede bifurcar y personalizar un artifact.
- Limitaciones indicadas: sin llamadas a APIs externas (en ese momento), sin almacenamiento persistente y API limitada a completions de texto.

## Recursos incluidos
- Skill: un brief reutilizable para explicar cómo funcionan los Claude-powered artifacts y qué limitaciones esperar.
