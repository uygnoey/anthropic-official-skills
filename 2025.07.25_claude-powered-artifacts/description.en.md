# Build and share AI-powered apps with Claude

## What is this post?
Introduces Claude-powered artifacts: interactive apps built and hosted inside the Claude app that can call Claude through an API, and can be shared so each user authenticates with their own Claude account and their usage is billed to their own subscription (no API keys to manage).

## When is it useful?
Useful when you want to prototype and share an AI-powered interactive experience quickly inside Claude without standing up infrastructure, or when explaining sharing and billing mechanics to teammates or users.

## Key points
- Artifacts can become interactive apps that call Claude via an API from within Claude.
- Shared apps use the viewer’s Claude account: their usage counts against their subscription; the creator doesn’t pay for their usage; no API keys are needed.
- Claude can generate and help debug real code; artifacts can be forked and customized.
- Limitations called out include no external API calls (at the time), no persistent storage, and a text-based completion API.
