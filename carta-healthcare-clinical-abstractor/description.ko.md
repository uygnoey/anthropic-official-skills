# Carta Healthcare가 임상 요약(추출) 담당자처럼 추론하는 AI를 만든 방법

## 이 글이 뭔가요
이 글은 Carta Healthcare가 Claude를 활용해 의료 기록을 추론하고, 대규모 환경에서 높은 정확도로 임상 데이터 추출을 수행하는 Lighthouse 플랫폼을 만든 과정을 설명합니다.

## 언제 유용한가요
지저분한 도메인 문서에서 근거를 들어 시간 범위를 지키며 판단해야 하고, 사람이 결과를 검토·신뢰해야 하는 AI 시스템을 설계할 때 유용합니다.

## 핵심 포인트
- 모델 성능만큼 컨텍스트 엔지니어링이 중요합니다. 어떤 근거를 포함/제외할지, 어떤 순서로 배치할지 결정하고, 환자별 컨텍스트를 런타임에 조립합니다.
- “수술 전(pre‑procedure)” 같은 조건에는 수술 시작 시각처럼 명시적 시간 기준점을 넣어 시간 논리를 정확히 적용하게 합니다.
- 평가 프레임워크를 일찍 만들고, 실패 원인이 프롬프트 문구인지/컨텍스트 누락인지/리트리벌 격차인지 식별할 수 있을 만큼 세분화합니다.
- 도메인 전문가(임상 추출 담당자)의 피드백을 프롬프트/컨텍스트 개선에 빠르게 반영합니다.
- 투명성을 우선합니다. 근거와 추론 이유를 함께 보여줘 사람이 검증할 수 있게 합니다.

## 번들 리소스
