## この記事について
この記事は、エージェント型コーディングをより安全かつより自律的にするための **ネイティブ・サンドボックス** に基づく Claude Code の新機能を2つ紹介します。研究プレビューとしてのサンドボックス化された bash ツールと、クラウド・サンドボックス上で動く Claude Code on the web が中心です。

## どんなときに役立つか
許可プロンプトを何度も出さずに Claude Code（または他のコーディングエージェント）により多くの操作を任せつつ、プロンプトインジェクションなどによる意図しないツール利用リスクを下げたいときに役立ちます。

## 主なポイント
- Claude Code は権限ベース（既定は読み取り専用）で動作しますが、頻繁な承認は「承認疲れ」を招き得ます。
- 記事は、操作ごとの許可よりもサンドボックスの方が安全で自律性が高いと述べます。
- 効果的なサンドボックスには **ファイルシステム隔離**（読み書き可能なディレクトリの制限）と **ネットワーク隔離**（接続可能なホストの制限）の両方が必要です。
- サンドボックス bash は OS のプリミティブ（例: Linux の bubblewrap、macOS の seatbelt）で制約を強制します。
- Claude Code on the web は各セッションをクラウド上の隔離サンドボックスで実行し、機密な認証情報をサンドボックス外に置いたまま、git 操作をプロキシ経由で処理します。
