# Admin rollout checklist (derived from the announcement)

Use this as a lightweight checklist when rolling out Claude Code in an organization, based on the capabilities described in the blog post.

## Seats
- Decide which users should have standard vs premium seats.
- Plan for onboarding and provisioning in the admin panel.

## Spend controls
- Set organization-level spend policies.
- Set per-user spend caps where needed for predictable billing.

## Policy settings
- Decide which tools should be permitted.
- Decide file access restrictions.
- Decide MCP server configurations to allow.

## Analytics
- Decide which adoption metrics to monitor (e.g., accepted lines of code, suggestion acceptance rate).

## Governance
- Plan how you will ingest Compliance API data into existing compliance/audit systems.
- Decide retention and selective deletion workflows.
