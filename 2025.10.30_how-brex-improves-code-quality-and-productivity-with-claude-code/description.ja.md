## この記事について
この記事は、Brex の各チームが Claude Code を日々の業務でどう活用しているか（エンジニアリングのレビュー・ワークフロー、コンテンツ／デザインシステムの保守、テキスト→SQL によるデータアクセスなど）を事例ベースで紹介します。

## どんなときに役立つか
組織全体で Claude Code の導入を進める際に、構造化したコンテキストファイル運用、CI/CD でのドキュメント更新チェック、コンテキスト対応のカスタムコマンドなど、具体的な定着パターンが必要なときに役立ちます。

## 主なポイント
- Claude Code は「レビューアのマインドセット」を可能にします。エンジニアは実装詳細よりも、方向性のガイドと変更のレビューに集中できます。
- 非技術職でも、簡単な文字列更新を PR で直接出したり、標準に照らしてデザインを自動チェックする Figma プラグインのようなツールを作れます。
- Claude Code と MCP サーバーを組み合わせ、テキスト→SQL インターフェースでデータアクセスを民主化できます。
- スケールのための運用パターンとして、ディレクトリごとの CLAUDE.md コンテキスト、ドキュメント更新を促す CI/CD チェック、/submit-pr のようなコンテキスト対応カスタムコマンドが挙げられます。

## 同梱リソース
- この記事の定着パターンを再利用可能な「Claude Code 組織導入プレイブック」（コンテキスト管理、ドキュメント衛生、カスタムコマンド）に落とし込んだ Agent Skill を 1 つ。
