## ¿De qué trata este post?
Este post explica cómo los equipos de Brex usan Claude Code en el trabajo diario: flujos de revisión de ingeniería, mantenimiento del sistema de diseño/contenido y acceso a datos mediante texto→SQL.

## ¿Cuándo es útil?
Es útil si vas a introducir Claude Code en toda una organización y necesitas patrones concretos de adopción, como archivos de contexto estructurados, comprobaciones de documentación en CI/CD y comandos personalizados con contexto.

## Puntos clave
- Claude Code habilita una “mentalidad de revisor”: los ingenieros guían la dirección y revisan cambios en lugar de centrarse en los detalles de implementación.
- Personas no técnicas pueden enviar cambios directamente (por ejemplo, crear PRs para actualizar cadenas simples) y construir herramientas como un plugin de Figma para revisar diseños contra estándares.
- Se puede democratizar el acceso a datos con una interfaz de texto→SQL impulsada por Claude Code y servidores MCP.
- Patrones de escalado destacados: archivos de contexto CLAUDE.md por directorio, comprobaciones en CI/CD que impulsan actualizaciones de documentación y comandos personalizados como /submit-pr que cargan el contexto relevante.

## Recursos incluidos
- Un Agent Skill que convierte estos patrones de adopción en un “playbook” reutilizable para desplegar Claude Code en una organización (gestión de contexto, higiene de documentación, comandos personalizados).
