# 이 스킬이 뭔가요
이 스킬은 GitHub 저장소를 연결한 뒤, 브라우저에서 코딩 작업을 위임하고(클라우드 격리 환경에서 실행) 결과를 PR로 정리하는 흐름을 돕습니다.

# 언제 써야 하나요
- 버그 수정, 사소한 개선, 의존성 업데이트처럼 범위가 명확한 작업이 있을 때
- 여러 저장소/여러 작업을 병렬로 처리하고 싶을 때
- 로컬 터미널 없이 진행 상황을 보면서 작업을 맡기고 싶을 때

# 핵심 원칙
- 작업을 작게 쪼개고(재현 → 수정 → 검증 → PR) 성공 기준을 명확히 합니다.
- PR 전에 테스트/검증을 통해 변경을 확인합니다.
- 진행 중 요구사항이 바뀌면 “steering” 방식으로 방향을 조정합니다.

# 예시
- “main 브랜치 CI가 깨져 있어요. 원인 찾아 고치고 PR 올려줘.”
- “deprecated 의존성 업데이트를 패키지별로 PR로 나눠 처리해줘.”

# 체크리스트·팁
- 저장소/브랜치/성공 기준(테스트 통과, 린트 등)을 먼저 전달합니다.
- 모호한 신규 개발보다는 잘 정의된 유지보수·버그픽스에 적합합니다.
