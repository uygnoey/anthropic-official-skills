# Skills로 프론트엔드 디자인을 개선하기: 실전 플레이북

이 가이드는 원문 글에서 제시한 프롬프트 구성/재사용 패턴을, 필요할 때만 불러 쓰는 Skill 형태로 정리한 것입니다.

## 문제: AI UI 결과의 ‘수렴’
원문은 UI 생성 결과가 흔한 폰트 스택과 예측 가능한 그라디언트 같은 ‘무난한 기본값’으로 수렴해 브랜드 개성이 약해질 수 있다고 설명합니다.

## 접근: 디자인 지침을 Skill로 분리해 재사용
- 디자인 지침을 (대개 마크다운) Skill 파일로 보관해 두고, 작업이 디자인 관련일 때만 로드합니다.
- 이렇게 하면 기본 프롬프트에 항상 넣어두느라 컨텍스트를 소모하지 않아도 됩니다.

## ‘적절한 고도’의 지시
- 너무 추상적: “모던하고 깔끔하게” 같은 지시는 결과를 충분히 바꾸기 어렵습니다.
- 너무 미시적: 헥스 코드/픽셀 단위까지 과도하게 고정하면 유연성이 떨어질 수 있습니다.
- 권장: 타이포그래피, 테마 레퍼런스, 모션, 배경 처리 등 ‘방향성’을 구체적으로 주되 구현은 맡깁니다.

## 원문에서 강조한 구체 영역
### 타이포그래피
- 흔한 기본 폰트는 피하라고 명시합니다.
- 더 개성 있는 폰트/조합을 선택합니다.
- 극단적인 굵기 대비와 확실한 스케일 점프를 활용합니다.

### 테마
원문은 RPG 테마 예시를 제공합니다:
- 판타지 풍 팔레트
- 장식적인 보더
- 양피지 느낌의 텍스처
- 드라마틱한 조명
- 중세풍 세리프 타이포

### 모션
- 필요하면 CSS 애니메이션과 스태거드 리빌을 사용합니다.
- React 작업에서는 모션 라이브러리를 활용할 수 있습니다.

### 배경
- 그라디언트/패턴을 테마 강화 용도로 의도적으로 사용합니다.
- 예측 가능한 기본값보다 기하학 패턴/텍스처 처리 등을 고려합니다.

## 웹 아티팩트 생성 관련 메모(원문 기반)
원문은 추가 Skill을 통해 React + Tailwind 및 컴포넌트 라이브러리 같은 현대적 도구를 이용해 더 풍부한 웹 산출물을 만들고, 이를 단일 HTML 아티팩트로 번들링하는 흐름을 언급합니다.
