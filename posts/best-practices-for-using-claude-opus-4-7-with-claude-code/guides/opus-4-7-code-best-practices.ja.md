# ガイド：Claude Code における Opus 4.7 のベストプラクティス

このガイドは、Claude Code でプロンプトやセッションを設定する際に活用できるチェックリスト形式で、記事の推奨事項をまとめたものです。

## 対話型セッションの構成
- Claude を逐次指示するペアプログラマーではなく、作業を委任できる優秀なエンジニアとして扱いましょう。
- 最初のターンでタスクを明確に提示してください。意図・制約・完了基準・関連ファイルの場所を含めます。
- 質問をまとめて送り、ユーザーとのやり取りの回数を減らすことで、余分な推論オーバーヘッドを抑えましょう。
- モデルが少ない確認でも安全に実行できると判断できる長時間タスクには、オートモードを適切に活用しましょう。

## 努力レベルのガイダンス
- `xhigh` は Claude Code における Opus 4.7 のデフォルトの努力レベルで、`high` と `max` の間に位置します。
- コストを抑えたい場合や複数のセッションを同時実行している場合は `high` を使用してください。
- `max` は本当に難しい問題にのみ意図的に使用してください。収益逓減が生じ、考えすぎになる可能性が高まります。

## 思考を促すプロンプト
- Opus 4.7 は、固定された思考バジェットを持つ Extended Thinking ではなく、適応型思考を使用します。
- より深い思考を促すには、回答前に慎重かつ段階的な推論を求めてください。
- 思考を抑えるには、疑問がある場合は素早く端的に回答するよう指示してください。

## 考慮すべき動作の変化
- 応答の長さはタスクの複雑さに応じて調整されます。特定のスタイルや長さが必要な場合は明示的に指定してください。
- モデルはツールの呼び出し頻度を下げ、より多く推論するようになります。ツールの使用を増やしたい場合は、いつ・なぜ使うべきかを明示的に説明してください。
- デフォルトではサブエージェントの生成数が少なくなります。ファイルや独立したタスクに対して並列処理が必要な場合は、明示的に並列サブエージェントをリクエストしてください。
