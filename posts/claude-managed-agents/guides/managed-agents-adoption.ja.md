# Claude Managed Agents 導入プレイブック

[Claude Managed Agents](https://claude.com/blog/claude-managed-agents) の 2026年4月8日付けローンチ発表をもとに編集しました。原文は製品発表ブログのため、本ガイドはその内容を意思決定・導入プレイブックとして再構成したものです。原文が提供していない再利用可能な開発者パターンは含みません。

## マネージドハーネスが必要な理由

Managed Agents 以前、プロダクション向けエージェントをリリースするには、エンジニアリングチームが最低限以下の項目を自前で構築または統合する必要がありました。

- サンドボックス化されたコード実行環境
- 長時間実行タスクを再開するためのチェックポインティング
- クレデンシャル管理
- 実システムに対するスコープ付きパーミッション管理
- エンドツーエンドのトレーシング
- モデルアップグレードのたびに作り直すエージェントループ

Managed Agents は**オーケストレーションハーネス**（ツール呼び出しのタイミング決定、コンテキスト管理、エラーからの回復方法を処理）とその運用インフラをセットで提供し、チームがプラットフォームではなくプロダクトを届けられるようにします。

## スイートに含まれる機能

| 機能 | 内容 | 提供状況 |
|---|---|---|
| プロダクショングレードのエージェント | セキュアなサンドボックス化、認証、ツール実行をすべて代行 | パブリックベータ |
| 長時間実行セッション | 数時間にわたって自律的に動作し、切断時も進捗と出力を保持 | パブリックベータ |
| マルチエージェント連携 | エージェントが他のエージェントを起動・指示して複雑な作業を並列処理 | リサーチプレビュー（アクセス申請が必要） |
| 信頼性の高いガバナンス | スコープ付きパーミッション、ID管理、実システムへの実行トレーシング | パブリックベータ |
| アウトカムベースモード | 成果と成功基準を定義するだけで、Claude が自己評価しながら反復実行 | リサーチプレビュー（アクセス申請が必要） |
| 従来のプロンプト＆レスポンス方式 | 細かい制御フローが必要なときに選択 | パブリックベータ |

## 意思決定チェックリスト：自社構築 vs. 導入

自前のエージェントランタイムを書き始める前に、以下の項目を確認してください。各項目は Managed Agents がすでに対応している事項に対応しています。

- [ ] サンドボックス分離、カーネルハードニング、脱出監視を担うチームがあるか？
- [ ] Claude モデルがアップグレードされるたびにエージェントループを再構築する準備があるか？
- [ ] 切断・デプロイ・フェイルオーバー時も長時間実行セッションを継続できるか？
- [ ] すべてのツール呼び出し・判断・障害を監査可能な形式でトレースする手段があるか？
- [ ] エージェントを実システムに対してファーストクラスのIDとして扱うスコープ付きパーミッションインフラがあるか？
- [ ] 上記すべてを構築したうえで、数カ月ではなく数週間でプロダクト価値を提供できるか？

いずれか一つでも「いいえ」であれば、マネージドランタイムを選ぶ根拠になります。

## 導入パス

原文で言及されている機能に沿ったステップバイステップの導入手順です。

1. **まず単一タスクランナーを選ぶ。** 原文の共通パターンは「1週間以内にデプロイされた専門エージェント」です。マルチエージェント連携ではなく、ここから始めてください。
2. **タスク・ツール・ガードレールを定義する。** それらがあなたの入力値であり、ハーネスがオーケストレーション・コンテキスト管理・エラー回復を担います。
3. **Claude Platform 上で実行する。** Claude Console、新しい CLI、または組み込みの `claude-api` Skill がある Claude Code を使います（`"start onboarding for managed agents in Claude API"`）。
4. **計測して検査する。** Claude Console のセッショントレーシング、インテグレーション分析、トラブルシューティング機能を使って、すべてのツール呼び出しと障害モードを確認してください。
5. **アウトカムベースモードの採用を判断する。** タスクに検証可能な成功基準がある場合は、アウトカムモードのリサーチプレビューアクセスを申請してください。そうでなければプロンプト＆レスポンス方式を維持してください。
6. **マルチエージェント連携は必要なときだけ追加する。** 並列専門エージェントが測定可能な価値をもたらす場合に、リサーチプレビューアクセスを申請してください。

## 計画に役立つ料金モデル

- 標準の Claude Platform トークンレート
- アクティブなランタイムに対して**セッション時間あたり $0.08** を追加

計画に織り込むべきコスト変数は2つです：1回の判断あたりのトークン消費量と、エージェントがアクティブ状態を維持する実経過時間。長時間自律セッションは短時間の集中実行よりもセッション時間コストが大きくなります。キャパシティプランニング時にそのトレードオフを考慮してください。
