# 배포 플레이북: 금융 서비스 AI 에이전트

[Building AI agents for financial services](https://claude.com/blog/building-ai-agents-in-financial-services) (Claude 블로그, 게시 2025-10-30)에서 정리한 상세 참고 자료. SKILL.md의 단계 목록보다 더 깊은 지침(통합 아키텍처, 단계별 롤아웃 계획, 리스크 통제 설계, 업계 사례)이 필요할 때만 이 파일을 읽어주세요.

## 단계별 롤아웃

### 1단계 — 시작점 찾기
이미 인간의 감독이 존재하거나, 불완전한 자동화의 결과가 작은 영역에서 **임팩트는 높고 리스크는 낮은** 후보를 고르세요. 좋은 후보:
- 고객 서비스 트리아지
- 내부 지식 조회
- 일상적 데이터 검증
- 이상 거래 패턴 플래깅
- 컴플라이언스 마감일 모니터링
- 문서 분류 자동화

단순한 과제 하나로 시작. 확대 전에 명확한 성공 지표를 정할 것.

### 2단계 — 조직 전체로 확장
일회성 솔루션보다 **재사용 가능한 기반**을 구축. 한 번 만든 문서 처리 능력이 다음 모두에 쓰일 수 있음:
- 은행 계정 대사, 인보이스 처리
- 컴플라이언스 팀의 규제 문서 분석
- 부서 간 금융 데이터 추출

신뢰 확보 방법: (a) 고객에게 AI vs 사람 여부를 투명하게 공개, (b) 직원 교육 — 에이전트 작동 원리와 에스컬레이션 시점 이해, (c) "대체"가 아닌 "보강"으로 포지셔닝.

### 3단계 — 복잡한 유즈케이스로 진입
기반이 안정된 뒤에만. 필수 요건:
- 모든 의사결정과 참조한 데이터 소스를 추적하는 포괄적 감사 로그
- 엣지 케이스 감지 실시간 모니터링
- 복잡한 케이스를 적절한 전문가에게 자동 라우팅하는 에스컬레이션 프로토콜
- 기술 정확도뿐 아니라 비즈니스 성과와 워크플로우 통합 성공까지 측정하는 지표

## 통합 및 아키텍처

### 레거시 인프라 도전 과제
- 벤더 간 코어 뱅킹 플랫폼 호환성 문제
- 부서별 데이터 사일로로 인한 크로스시스템 오케스트레이션 필요
- 레거시 메인프레임 통합 난이도
- 실시간 동기화 요구

### 통합 접근 방식
1. **직접 통합** — 에이전트 플랫폼이 대상 시스템을 네이티브로 지원할 때
2. **커스텀 커넥터** — API 또는 MCP 방식
3. **미들웨어** — 트랜잭션 무결성과 감사 추적을 유지하며 시스템 간 격차를 메움

### 에이전트형 데이터 흐름
에이전트는 서로 무관한 여러 소스에서 정보를 수집하고 이질적 데이터 유형(거래 기록, 시장 데이터, 규제 문서)을 처리함. 원문 예시: 분석가가 5개 시스템에서 수동으로 데이터를 뽑는 대신, 에이전트가 그 시스템들을 가로질러 거래 패턴을 모니터링하고 근거 있는 권고를 제시.

### 재사용 가능한 기반
여러 부서에 걸쳐 쓰이는 공유 인프라를 구축. 발전하는 AI 역량과 함께 진화할 수 있는 모듈형 시스템을 선호할 것.

## 리스크 통제

실시간 리스크 구현에 반드시 포함해야 할 요소:
- **투명한 추론** — 금융 전문가가 검증하고 고객에게 설명 가능해야 함
- **명확한 에스컬레이션 경로** — 복잡하거나 모호한 상황 대응
- **오버라이드 기능** — 고객 사정에 따라 어드바이저가 AI 권고를 거부할 수 있어야 함
- **페일세이프 기본값** — 운영 효율보다 고객 보호 우선
- **고위험 액션에 대한 인간 승인(human-in-the-loop)**
- **장애 시 알려진 안전 상태로 수렴**

초기 배포에서 인간 판단을 항상 루프 안에 유지. 고객에게 AI 상호작용 공개.

## 규제 및 컴플라이언스

에이전트는 각 의사결정에 **어떤 규제 프레임워크가 적용되는지**, 그리고 **다양한 감사 요구사항에 맞춰 어떻게 기록할지**를 알아야 함. 원문에 언급된 기관: SEC, FDIC, 주(州) 은행 당국, 국제 기구들.

특정 컴플라이언스 고려 사항:
- AI 데이터 처리 워크플로우에 대한 SOC 2 및 PCI DSS 준수
- 리스크 평가 정확도에 대한 증거 기반 검증
- AI 의사결정 감사 추적 문서화 요구
- 에이전트형 솔루션에 처음부터 관찰 가능성과 추적성을 내장

## 의사결정 체크리스트

배포 승인 전 확인:
- [ ] 이 워크플로우에 이미 인간 감독이 존재하는가?
- [ ] 불완전한 자동화의 결과가 수용 가능한 수준인가?
- [ ] 명확한 성공 지표가 정의되어 있는가?
- [ ] 에이전트가 필요한 시스템과 직접 통합되는가? 아니면 커넥터/미들웨어 경로가 예산에 반영되어 있는가?
- [ ] 의사결정 유형별로 적용 규제 프레임워크가 식별되었는가?
- [ ] 데이터 처리에 대한 SOC 2 / PCI DSS 이슈가 해결되었는가?
- [ ] 모든 의사결정과 데이터 소스에 대한 감사 추적이 캡처되는가?
- [ ] 투명한 추론, 에스컬레이션, 오버라이드, 페일세이프 기본값이 설계에 포함되었는가?
- [ ] 고위험 액션에 대한 인간 승인 지점이 있는가?
- [ ] 고객 대면 용도라면: AI 상호작용이 공개되고, 사람으로 가는 경로가 있는가?
