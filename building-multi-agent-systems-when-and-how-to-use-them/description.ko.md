## 이 글이 뭔가요
이 글은 단일 에이전트보다 멀티 에이전트 시스템이 더 잘 작동하는 상황과, 안전하게 작업을 분해·검증하는 방법을 설명합니다.

## 언제 유용한가요
단일 에이전트가 컨텍스트 오염으로 성능이 떨어지거나, 병렬 탐색이 필요하거나, 전문화로 신뢰성을 높여야 할 때 유용합니다.

## 핵심 포인트
- 단일 에이전트부터 시작하고, 멀티 에이전트는 토큰 비용과 복잡도가 커진다는 점을 고려합니다.
- 멀티 에이전트는 주로 컨텍스트 보호, 병렬화, 전문화에 사용합니다.
- 문제 “단계”가 아니라 컨텍스트 경계 기준으로 분해하는 접근(컨텍스트 중심 분해)을 선호합니다.
- 검증 서브에이전트 패턴을 사용하되, “성급한 승리(early victory)”를 막기 위해 명시적 기준을 둡니다.

## 번들 리소스
