## 이 글이 뭔가요
이 글은 에이전트 워크플로의 대표 패턴 3가지(순차, 병렬, 평가자–최적화)와 각 패턴을 언제 선택해야 하는지 설명합니다.

## 언제 유용한가요
여러 단계로 구성된 에이전트 시스템을 설계할 때, 지연 시간·비용·신뢰성의 트레이드오프를 고려해 실행 패턴을 고르고 싶을 때 유용합니다.

## 핵심 포인트
- 먼저 단순하게: 단일 에이전트로 가능한지 테스트한 뒤, 필요할 때만 워크플로로 확장합니다.
- 순차 워크플로는 단계 간 의존성이 있을 때 적합하지만 지연이 늘어납니다.
- 병렬 워크플로는 독립 서브태스크를 동시에 처리해 시간을 줄이지만, 결과를 합치는 전략이 필요하고 비용이 늘 수 있습니다.
- 평가자–최적화는 반복 개선으로 품질을 올리지만 토큰 사용량이 증가하며 명확한 종료 조건이 필요합니다.

## 번들 리소스
- 의사결정 체크리스트와 패턴 설명을 Agent Skill로 정리했습니다.
