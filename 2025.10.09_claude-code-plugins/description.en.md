## What is this post?
This post introduces Claude Code plugins: a lightweight way to package and share Claude Code customizations that can be installed and toggled on/off within Claude Code.

## When is it useful?
Use plugins when you want to standardize or share a Claude Code setup (e.g., team workflows, project-specific utilities) without manually copying configuration between environments.

## Key points
- Plugins can bundle any combination of extension points: slash commands, subagents, MCP servers, and hooks.
- Plugins are designed to be enabled only when needed to reduce prompt context and complexity.
- Plugins can be distributed through marketplaces.
- Plugins can be installed from inside Claude Code via the `/plugin` command (public beta).
