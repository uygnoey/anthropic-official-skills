# このスキルについて
このスキルは、GitHub に接続したブラウザベースのワークスペースからコーディングタスクを委任し、各タスクを隔離されたクラウド環境で実行してプルリクエストとして出力する流れを支援します。

# どんなときに役立つか
- 明確に定義されたバグ修正やルーティンなコードメンテナンスがあるとき。
- 複数のリポジトリにわたって複数のコーディングタスクを並行して実行したいとき。
- ローカルターミナルを使わず、ブラウザでタスクの進捗を確認しながら作業を進めたいとき。

# 基本原則
- タスクをスコープを絞り、検証可能な状態に保つ（明確な受け入れ基準を設ける）。
- PR を作成する前に変更を検証する（テスト・lint など）。
- ステアリングを活用する：タスクの進行中に要件を調整する。

# 主なポイント（例）
- 失敗しているテストスイートを修正し、PR を作成する。
- バックログの小さな issue を個別の PR として処理する。

# チェックリスト / ヒント
- リポジトリ・ブランチ・成功基準を最初に伝える。
- 曖昧なゼロからの開発よりも、ルーティンな修正作業を優先する。
