# Claude Sonnet 4 が 100万（1M）トークンのコンテキストに対応

## この記事について
この記事は、Anthropic API において Claude Sonnet 4 が最大100万（1M）トークンのコンテキストをサポートしたことを発表し、それによって可能になることを要約します。

## どんなときに役立つか
1回のリクエストで非常に大きな入力（例：コードベース全体、膨大な文書セット）を分析したい場合や、長い多段のワークフロー全体で一貫性を保つエージェントを構築したい場合に役立ちます。

## 主なポイント
- Sonnet 4 は最大1Mトークンのコンテキストに対応（5倍増）。
- 主なユースケースは、大規模コード解析、文書の統合（シンセシス）、コンテキスト認識エージェント。
- 長文コンテキストは Claude Developer Platform でパブリックベータ提供され、Amazon Bedrock と Google Cloud Vertex AI でも利用可能。
- 200Kトークン超のプロンプトには価格変更があり、プロンプトキャッシュやバッチ処理で遅延/コストを下げられます。

## 同梱リソース
- 長文コンテキストを使うべき場面と、何を入れるかを判断するための短い Claude Code スキル
